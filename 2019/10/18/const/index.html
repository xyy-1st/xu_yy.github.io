const 修饰变量

const关键字可以限定一个变量的值 只读 不允许改变

功能一样,都是声明一个只读的常量a.且a必须初始化,之后值不能改变,但可以通过指针来改变。

功能一样，与上面相同，const都是修饰 *a 的。
效果：a的值可以改变，*a的值不能改变 但可以通过”别名”来改变

这个const 是修饰d的,必须初始化
效果：d的值不能改变，*b的值可以改变

是const int *p 与int * const p 的结合，必须初始化
效果： p与 *p 都不能改变

const 与宏定义的区别

(1) 编译器处理方式不同

　　define宏是在预处理阶段展开。

　　const常量是编译运行阶段使用。

(2) 类型和安全检查不同

　　define宏没有类型，不做任何类型检查，仅仅是展开。

　　const常量有具体的类型，在编译阶段会执行类型检查。
(3)const 可以节省空间，避免不必要的内存分配

定义常量时最好使用const
但是定义矩阵的时候要使用宏定义，因为宏定义（预处理）比const（编译）先处理
